# Positioning Canvas (Step 10)

This skill produces the completed positioning canvas from April Dunford's framework and prepares it for company-wide use.

## What this step produces

The positioning canvas is a single-page document that captures all five (plus one) components of positioning in a structured format that can be shared with marketing, sales, product, and customer success. It is the output that makes positioning real inside a company — without it, the work done in Steps 1–9 stays in the heads of a few people and erodes immediately.

The canvas is not a messaging document. It is the source of truth that messaging is derived from. It is not a tagline. It is the strategic foundation that informs taglines.

## Inputs required

To build a complete canvas, the skill needs:

1. **Product name and one-line description** — what it is in plain language
2. **Market category** (and subcategory if applicable) — from Step 8
3. **Competitive alternatives** — from Step 4, ranked by frequency for best-fit customers
4. **Unique attributes** — from Step 5, the things you have that alternatives lack
5. **Value** — from Step 6, the provable customer outcomes those attributes enable
6. **Who cares a lot** — from Step 7, the customer characteristics that predict high affinity
7. **Relevant trend** (optional) — from Step 9, if one applies

If any of these are missing, the skill will ask for them or flag that the corresponding earlier step has not been completed.

## How to build the canvas

### Collect and validate inputs

Before assembling the canvas, validate that each component is complete and internally consistent:

- **Competitive alternatives** should reflect customer perspective, not founder perspective
- **Unique attributes** should be compared against those alternatives specifically
- **Value statements** should be provable, not aspirational
- **Target market characteristics** should be specific and identifiable, not demographic generalities
- **Market category** should be something customers recognize and that triggers favorable assumptions

If any component is weak, the canvas will flag it. It is better to fix gaps here than to distribute positioning that will not hold up in sales conversations.

### Assemble the canvas

Use this structure:

```
POSITIONING CANVAS

Product: [name]
One-line description: [what it does in plain language]

Market category: [primary category]
Subcategory (if applicable): [more specific segment]

Competitive alternatives (ranked by frequency):
1. [most common alternative]
2. [second most common]
3. [etc.]

┌─────────────────────┬─────────────────────┬───────────────────────┬─────────────────────┐
│ Competitive         │ Unique attributes   │ Value                 │ Who cares a lot     │
│ alternative         │ vs. this alternative│ (customer outcomes)   │ (characteristics)   │
├─────────────────────┼─────────────────────┼───────────────────────┼─────────────────────┤
│ [primary            │ [what we have that  │ [what customers can   │ [what makes a       │
│ alternative]        │ this lacks]         │ do because of this]   │ customer care       │
│                     │                     │                       │ deeply]             │
├─────────────────────┼─────────────────────┼───────────────────────┼─────────────────────┤
│ [secondary          │ [attributes vs.     │ [value delivered      │ [who needs this     │
│ alternative]        │ this one]           │ here]                 │ value most]         │
└─────────────────────┴─────────────────────┴───────────────────────┴─────────────────────┘

Relevant trend (if applicable):
[Trend name] — [one sentence on how it connects to this positioning and why it adds urgency]
```

### Quality checks before sharing

Before finalizing, run these checks:

**Consistency check:** Do the attributes in the canvas match the competitive alternatives they are compared against? Attributes are only unique relative to specific alternatives.

**Proof check:** Are value claims backed by evidence? Flag any that are aspirational rather than demonstrated. Mark them [VALIDATE].

**Simplicity check:** Can a new hire read the canvas and understand what the product is, who it's for, and why it's better? If not, simplify.

**Sales readiness check:** Would a salesperson know what to say in the first two minutes of a call based on this canvas? If not, the who-cares column or the value column needs to be sharpened.

**Alignment check:** If this canvas were shown to the 5–10 best-fit customers from Step 1, would they recognize themselves in it? If not, something is off.

## What to build after the canvas

The canvas feeds three immediate downstream work streams:

### 1. Sales story (most urgent for companies with human sales)

The canvas provides the "what." The sales story provides the "why it matters to you." Route to the `sales-story-builder` skill to translate the canvas into a narrative arc for first calls.

### 2. Messaging document

The canvas is the foundation; a messaging document applies it to specific channels and audiences. The marketing team should write a master messaging document anchored to the canvas rather than creating messaging from scratch for every campaign.

Key principle: every piece of marketing copy starts from the canvas. Without this anchor, messaging creeps over time as writers build on modified messages in a chain, slowly drifting from the positioning.

A minimal messaging document includes:
- **Headline** — the sharpest possible single statement of value, framed in the chosen market category
- **Subheadline** — one sentence expanding on the headline, addressing the primary competitive alternative
- **Three value theme statements** — one paragraph per theme, written for the primary buyer
- **Proof points** — one piece of evidence per value theme
- **Buyer description** — who this is for, in the language of the "who cares a lot" column

### 3. Product roadmap and pricing

Changing the positioning often reveals misalignments between the canvas and the product roadmap. Features that don't reinforce the value themes should be questioned. Features that would strengthen the position should move up.

Similarly, pricing should reflect the chosen market category. After repositioning from one category to another, pricing often needs adjustment — both the model and the level.

## Output format

Deliver:

1. The completed positioning canvas (structured table format)
2. Any flagged gaps or weaknesses, with specific recommendations to address them
3. A brief distribution note: who in the company should receive this canvas and in what form
4. A checklist of next steps (sales story, messaging document, roadmap review, pricing review)

Keep the canvas to a single page. If it requires more, the components are too detailed — summarize and create reference documents for the detail.
